ECO Change Detection PoC

반도체 공정 변경점(ECO) 적용 전후의 품질 차이를
3종 Score로 자동 정량화하고, 주요 원인 Feature를 식별합니다.

3-Score Pipeline: Shift Score | Tail Score | Outlier Wafer Score
1. 과제 배경 2. 실험 과정 3. 결과 4. 토론 5. 인사이트 6. 대시보드 7. 참고 문헌

01 / 과제 배경

1.1 목적

반도체 공정에서 ECO(Engineering Change Order)를 적용한 후, 변경 전(Ref)과 변경 후(Compare)의 품질이 동일한지를 객관적으로 검증해야 합니다.

현업에서는 수천 개의 EDS/MSR/AWACS Feature를 엔지니어가 수작업으로 비교하며, 이 과정에서 미세한 변화나 간헐적 이상이 누락될 위험이 있습니다. 본 파이프라인은 이 과정을 자동화하여, 변경점 적용의 안전성을 정량적으로 평가합니다.

1.4 설계 원칙

왜 3종 Score인가?
  • 단일 Score로 합치지 않는다: 불량 패턴이 2종류(Systematic Shift vs Intermittent Spike)이므로, 각각에 맞는 Score를 별도 산출
  • 3개 Score를 OR 조건으로 판정: 하나라도 기준 초과 → 경고 상향
  • Score별 원인 Feature를 따로 추적: "전반적 drift"와 "간헐 불량"은 다른 엔지니어링 액션으로 이어지기 때문

02 / 실험 과정

2.1 파이프라인 구조

1
전처리
결측/상수 변수 제거 → Ref 기준 Robust Scaling (median/IQR) → Winsorizing (0.5th~99.5th)
Robust Scaling을 쓰는 이유: Mean/Std는 극단값에 취약. Median/IQR은 heavy-tail 분포가 흔한 반도체 공정 데이터에 적합.
2
3종 Score 산출 (병렬)
Shift Score (중심치 이동) | Tail Score (간헐적 극단값) | Outlier Wafer Score (wafer 집중 이상)
3
Feature Importance 추적
Score별 원인 Feature Top-N 식별 (Shift: 방향 포함, Tail: 초과율, Outlier: 공통 Feature)
4
판정 + 리포트
SAFE / CAUTION / RISK / HIGH_RISK 자동 판정 + 시각화 리포트
파이프라인 흐름도
[해석] 입력 데이터가 전처리를 거친 후, 3종 Score가 병렬로 산출됩니다. 각 Score는 서로 다른 불량 패턴을 탐지하도록 설계되어 있으며, Feature Importance 추적과 최종 판정이 순차적으로 이루어집니다.

2.2 3종 Score 설계

Score 1 — Shift Score (중심치/산포 이동)

zj = (meancompare,j - meanref,j) / stdref,j    →    Shift Score = mean(|zj|) for Top-K features

Top-K 방식의 이유: 전체 RMS는 5,000개 feature에서 소수 feature의 shift가 희석됩니다. 상위 K개(1%)만 보면 실제 shift한 feature가 Score에 직접 반영됩니다.

Score 범위해석
≈ 0Ref와 Compare 거의 동일
> 0.5소폭 이동 관찰 → CAUTION
> 1.0상위 feature 평균 1σ 이상 이동 → RISK
> 2.0명확한 systematic shift → HIGH_RISK

Score 2 — Tail Score (간헐적 극단값)

thresholdj = ref의 99th percentile    →    tail_ratej = P(comparej > thresholdj)

현장에서 중요한 이유: 수율은 괜찮아 보이는데 간헐적으로 불량 lot이 나오는 상황. Mean 기반 z-shift로는 절대 안 잡힙니다. Ref에서 정의상 이 비율은 약 1%이므로, Compare에서 3% 이상이면 유의미한 tail 증가입니다.

Score 범위해석
< 0.03tail 증가 없음 → SAFE
0.03 ~ 0.10간헐적 이상 발생 → CAUTION ~ RISK
> 0.1010%+ wafer가 극단값 초과 → HIGH_RISK

Score 3 — Outlier Wafer Score (wafer 단위 이상)

exceed_ratiow = (threshold 초과 feature 수) / (전체 feature 수)    →    Outlier = exceed_ratio > 5%

핵심 아이디어: 간헐적 불량은 특정 wafer에 집중되는 경우가 많습니다. Feature 단위로 보면 각각은 미미해도, wafer 단위로 보면 하나의 wafer가 수십 개 feature에서 동시에 튀는 패턴을 잡습니다.

2.3 합성 데이터 설계 (5가지 패턴)

실제 데이터 투입 전 파이프라인 동작을 검증하기 위해, 3단계 난이도로 구성된 5가지 불량 패턴을 삽입했습니다.

기저 데이터: Ref 1,000 wafers × 5,000 features, Compare 80 wafers × 5,000 features (N(3.0, 0.5²))

Pattern A: Systematic Shift Easy
F100~120 (21개), +1.5σ, 전체 wafer에 적용
→ Shift Score가 검출해야 함
Pattern B: Intermittent Spike Easy
F500~510 (11개), 10% wafer에서 +6σ spike
→ Tail Score가 검출해야 함
Pattern C: Multi-Feature Outlier Easy
F200~499 (300개), W70~74에서 +4σ 동시 이상
→ Outlier Wafer Score가 검출해야 함
Pattern D: Gradual Trend Medium
F600~610 (11개), 0→+2σ 점진적 drift
→ 시계열 특성, 현 파이프라인 한계 테스트
Pattern E: Subtle Shift Hard
F700~720 (21개), +0.5σ 미세 이동
→ 검출 한계 테스트 (Top-K 임계값 미달 예상)
합성 데이터 구조 분석 — 5가지 불량 패턴
[해석]
  • 정상 Feature(EDS_0050): Ref와 Compare의 분포가 완전히 일치합니다. 변화 없는 feature는 차이가 없어야 합니다.
  • Pattern A(EDS_0110): Compare의 평균이 Ref 대비 명확히 우측으로 이동(+1.5σ). 빨간 점선(Comp mean)이 파란 점선(Ref mean)보다 우측에 위치합니다.
  • Pattern B(EDS_0505): 대부분 wafer는 정상이나, 소수 wafer에서 극단값이 발생합니다. 히스토그램의 우측 꼬리가 길어진 것이 특징입니다.
  • Pattern D(EDS_0605): wafer 순서에 따라 점진적으로 상승하는 추세가 보입니다. 빨간 점선은 선형 회귀 추세선으로, 양의 기울기가 drift를 나타냅니다.

3.2 전처리 결과

전처리 과정: 필터링 → Robust Scaling → Winsorizing
[해석]
  • (좌상) Feature 필터링: 5,000개 전체 feature가 유효. 합성 데이터에 결측/상수가 없어 제거 대상 없음.
  • (중상/우상) Scaling 전후: 원본은 동일 스케일(평균 3.0)이지만, Scaling 후 Ref의 median=0, IQR≈1로 정규화. 이를 통해 Compare의 이탈이 σ 단위로 해석 가능.
  • (좌하) Scaling 검증: Ref의 Median≈0, IQR≈1에 집중되어 Robust Scaling이 정상 작동.
  • (중하) EDS_0110 비교: Scaling 후 Compare가 Ref 대비 우측으로 ~1.5σ 이동. Pattern A가 전처리 후에도 명확히 보존.
  • (우하) Winsorizing 효과: EDS_0505의 극단값(spike)이 Winsorizing 후 제한되지만, 완전히 제거되지는 않아 Tail Score에서 여전히 검출 가능.

3.3 Score 산출 상세

3종 Score 산출 과정 — 각 Score의 산출 근거와 개념도
[해석]
  • (1행 — Shift Score)
    • 좌: Feature별 z-shift 분포. 대부분 0 근처이나, 빨간색(+0.5σ 이상) feature들이 Pattern A 영역에서 돌출.
    • 중: Top-15 전부 EDS_01xx (F100~120). Pattern A가 100% 정확히 검출됨.
    • 우: 개념도 — Ref 분포(파랑)에서 Compare 분포(빨강)가 1.5σ 우측으로 이동한 모습.
  • (2행 — Tail Score)
    • 좌: Feature별 tail rate 분포. 상위 feature들이 10% 위험선을 대거 초과. 661개 feature가 기대치(1%)의 3배 이상.
    • 중: Elevated feature 상위 15개의 tail rate. 최대 31.25%로, Ref 99th 이상 구간에 Compare의 약 1/3이 포함.
    • 우: 개념도 — 정상 분포의 우측 꼬리(빨간 영역 ~1%)가, Compare에서 heavy tail로 대폭 확대된 모습.
  • (3행 — Outlier Wafer Score)
    • 좌: Wafer별 초과 비율. 5개 wafer(comp_w0070~0074)만 5% 기준을 크게 초과하여 명확히 구분.
    • 중: Outlier wafer들의 공통 이상 Feature. F200~499 범위의 feature들이 높은 비율로 동시 초과.
    • 우: 3종 Score 요약 박스. OR 조건으로 최종 HIGH_RISK.

3.4 Feature Importance

Feature Importance: 3종 Score별 원인 Feature 분석
[해석]
  • (좌상) Shift Top-20: 전부 EDS_0100~0120 범위. 모두 '악화' 방향(빨간색)이며 z-shift +1.2~1.6σ. Pattern A의 feature가 정확히 식별됨.
  • (우상) Tail Top-20: EDS_02xx~04xx 범위(Pattern C 영역). 5개 outlier wafer에서 이 feature들이 대거 초과했기 때문. Pattern B(F500~510)도 포함.
  • (좌하) Shift vs Tail Scatter: Feature들이 4개 영역으로 분류됨. 빨간점(Shift+Tail)은 교차 영향, 주황(Shift만)은 Pattern A, 보라(Tail만)은 Pattern B/C. 대부분(회색)은 정상.
  • (우하) Multi-Score Heatmap: 상위 feature의 3종 Score 기여도. 각 Score에서 서로 다른 feature가 top에 와서, 3종 Score의 상호 보완성이 시각적으로 확인됨.

최종 판정 대시보드 — Score 요약 + 원인 Feature + Outlier Wafer + 권장 액션
[해석] HIGH_RISK 판정과 함께, 3가지 사유가 명확히 제시됩니다:
  • Shift Score 0.99: 상위 feature에서 유의미한 이동 (Pattern A)
  • Tail Score 31.2%: 심각한 tail 증가 (Pattern B + C)

3.6 민감도 검증

민감도 분석 — False Alarm, 단조 증가, Sample Size, 체크리스트
[해석]
  • (좌상) False Alarm Test: Ref를 반으로 나누어 비교 시 모든 Score ≈ 0. 변화 없는 데이터에서 오탐이 발생하지 않음을 확인.
  • (우상) 단조 증가 검증: Shift 크기를 0→2.0σ로 늘렸을 때, Score가 단조 증가. Score가 이상의 크기에 비례적으로 반응함을 검증.
  • (좌하) Sample Size 검증: Compare 30장 미만에서 Score 변동성(error bar) 급증. 최소 30장 이상의 Compare가 필요하다는 운영 기준의 근거.
  • (우하) 체크리스트: Pattern A/B/C 검출, False Alarm, 단조 증가 — 5개 항목 모두 PASS.

3.7 추가 인사이트 시각화

다양한 시각화 기법: Violin Plot, 2D Scatter, Correlation Heatmap, CDF 비교
[해석]
  • (좌상) Violin Plot: Top-5 Shift Feature에서 Ref(파랑)와 Compare(빨강)의 분포 형태와 중앙값 차이가 한눈에 보입니다. 모든 feature에서 Compare가 일관되게 우측으로 이동.
  • (우상) 2D Scatter: 상위 2개 Shift Feature의 2차원 공간. Ref(파랑)와 Compare(빨강) 클러스터가 분리되어 다변량 관점에서도 shift 확인.
  • (좌하) Correlation Heatmap: 상위 feature 간 상관관계가 ≈0으로 낮음. 각 feature의 shift가 독립적이며, 만약 높았다면 소수의 근본 원인이 파생된 것으로 해석 가능.
  • (우하) CDF 비교: KS test로 두 분포의 차이를 통계적으로 검증. KS statistic이 높고 p-value가 극히 낮아 통계적으로 유의미한 차이.

3.8 Ground Truth 검증

Ground Truth 대비 검출 성능 — 패턴별 Recall, 종합 Precision/Recall, 분포 비교
[해석]
  • (좌상) 패턴별 Recall: Easy(A, B, C)는 높은 검출률. Medium(D)과 Hard(E)는 상대적으로 낮아, 파이프라인의 검출 한계를 정량적으로 보여줍니다.
  • (중상) 종합 Precision/Recall: 높은 Precision은 "검출한 것이 실제 이상일 확률"이 높다는 의미. 오탐을 최소화하면서 주요 패턴은 검출.
  • (우상) Score별 TP: Shift/Tail/Outlier가 각각 다른 패턴의 feature를 검출하여 3종 Score의 상호 보완성 입증.
  • (하단) 분포 비교: Pattern A(명확 shift), D(점진적 trend), E(미세 shift)의 실제 분포. Pattern E의 z-shift ≈ 0.5σ로, 현재 Top-K 방식의 검출 한계.

3.9 Wafer 단위 심층 분석

Wafer 단위 분석: 이상 비율 분포, Outlier Profile, Normal vs Outlier 비교
[해석]
  • (좌) 이상 비율 분포: 대부분의 wafer는 1~2% 수준으로 정상이지만, 5개 outlier wafer만 5%를 크게 초과하여 명확한 bimodal 분포를 형성.
  • (중) Outlier Feature Profile: 5개 outlier wafer(comp_w0070~0074)의 상위 feature 프로파일이 유사한 패턴. 동일 원인에 의한 이상임을 시사하며, 현장에서 해당 wafer의 공정 이력(설비, 시간, lot)을 역추적하는 근거.
  • (우) Normal vs Outlier: Outlier wafer 평균이 Normal 대비 일관되게 높아, 다변량 관점에서의 이상이 확인됨. 단일 feature가 아닌 다수 feature 동시 이상이 특징.

04 / 토론 (Discussion)

4.1 3종 Score 설계의 타당성

본 파이프라인은 단일 Score가 아닌 3종 Score를 병렬로 산출하는 설계를 채택했습니다. 실험 결과, 각 Score가 서로 다른 불량 패턴을 검출하는 것이 확인되었습니다:

Score검출한 패턴다른 Score로 검출 가능?
Shift ScorePattern A (Systematic Shift)Tail로는 부분적, Outlier로는 불가
Tail ScorePattern B (Spike) + C (Outlier)Shift로는 불가 (평균에 희석)
Outlier ScorePattern C (Multi-feature)개별 feature 분석으로는 미미
만약 단일 Score를 사용했다면?
Pattern B(간헐적 spike)는 평균 기반 지표에 희석되어 검출 어려움. Pattern C는 feature 단위로는 개별 이상이 미미하여 놓칠 가능성. 3종 Score의 상호 보완성이 실험적으로 입증되었습니다.

4.2 Top-K 방식의 장단점

측면내용
장점5,000개 feature 중 소수만 shift한 경우에도 민감하게 반응. 전체 RMS 대비 noise에 강건.
한계K의 선택에 따라 Score 변동. K 너무 작으면 noise에 취약, 너무 크면 signal 희석.
본 실험K=1%(50개)에서 Pattern A의 21개 feature가 Top-50에 모두 포함 → 적절한 설정.

4.3 난이도별 검출 한계

어떤 패턴을 잡지 못하는가?
  • Pattern D (Gradual Trend, Medium): 부분 검출. 시간에 따른 점진적 변화는 wafer 순서를 고려하지 않는 현재 파이프라인으로는 전체 평균에서만 부분적으로 검출. → CUSUM이나 시계열 기반 방법 도입 필요.
  • Pattern E (Subtle Shift, Hard): 미검출. +0.5σ 수준의 미세 변화는 Top-K 임계값 미달. → PCA 기반 다변량 분석이나 Mann-Whitney U 통계 검정 보완 필요.

4.4 False Alarm vs Sensitivity Trade-off

민감도 분석에서 확인된 핵심 사항:

  • False Alarm = 0: Ref vs Ref 비교 시 모든 Score ≈ 0. 오탐이 발생하지 않음.
  • 단조 증가: 이상 크기에 비례하여 Score 증가. 임계값 조정으로 민감도 튜닝 가능.
  • 최소 Sample Size = 30: Compare 30장 미만에서 변동성 급증. 운영 정책에 반영 필요.

4.5 Robust Scaling의 효과

측면Robust Scaling (median/IQR)Standard Scaling (mean/std)

05 / 인사이트 (Key Insights)

5.1 실무 적용 시사점

Insight 1: 3종 Score 병렬 산출은 필수적
단일 Score로는 불량 패턴의 다양성을 포착할 수 없습니다. 특히 간헐적 spike(Pattern B)는 평균 기반 지표로는 검출 불가능합니다. Shift Score만 사용했다면, 10% wafer에서 +6σ spike가 발생해도 전체 평균에서 희석되어 놓쳤을 것입니다.
Insight 2: Feature Importance의 이원화가 핵심 가치
Shift 원인과 Tail 원인이 다를 때가 더 중요합니다. 전반적 drift(공정 조건 조정 필요)와 간헐 불량(설비 점검 필요)은 다른 공정 액션으로 이어지기 때문입니다. 본 실험에서도 Shift Top(EDS_01xx)과 Tail Top(EDS_02xx~05xx)이 완전히 다른 feature 세트였습니다.
Insight 3: Outlier Wafer 공통 Feature = 역추적 근거
다수 feature에서 동시에 이상이 발생하는 wafer를 식별하면, 특정 설비/시간대/lot의 문제를 역추적할 수 있습니다. 본 실험의 5개 outlier wafer(W70~74)가 유사한 Feature Profile을 가진 것은, 동일 원인에 의한 이상임을 시사합니다.
Insight 4: Compare 최소 30장 = 운영 정책 근거
Sample size 30장 미만에서 Score 변동성이 급증합니다. 이 정량적 근거를 통해, 현업에서 "Compare 30장 이상 확보 후 분석 실시"라는 운영 정책을 수립할 수 있습니다.
