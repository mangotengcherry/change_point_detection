파이프라인 흐름도

합성 데이터: 5가지 불량 패턴

3단계 난이도로 구성된 합성 데이터로 파이프라인의 검출 능력을 검증합니다.

Pattern A: Systematic Shift Easy
F100~120 (21개), +1.5σ 이동, 전체 wafer
Pattern B: Intermittent Spike Easy
F500~510 (11개), 10% wafer에서 +6σ spike
Pattern C: Multi-Feature Outlier Easy
F200~499 (300개), W70~74에서 +4σ 동시 이상
Pattern D: Gradual Trend Medium
F600~610 (11개), 0→+2σ 점진적 drift
Pattern E: Subtle Shift Hard
F700~720 (21개), +0.5σ 미세 이동
합성 데이터 구조 분석

Step 1: 전처리

Ref 기준 Robust Scaling으로 Feature별 스케일 차이를 정규화하고, Winsorizing으로 극단적 측정 오류를 제거합니다.

전처리 과정: 필터링 → Robust Scaling → Winsorizing

Step 2: 3종 Score 산출

Shift Score: 중심치/산포 이동 탐지

각 Feature별로 Compare의 평균이 Ref 대비 몇 σ 이동했는지 Z-shift를 계산합니다. 상위 Top-K Feature의 |Z-shift| 평균이 최종 Score입니다.

Tail Score: 간헐적 극단값 탐지

Ref의 99th percentile을 기준으로, Compare에서 이를 초과하는 비율(Tail Rate)을 Feature별로 측정합니다. 기대치(1%)를 크게 상회하면 간헐적 이상이 발생한 것입니다.

Outlier Wafer Score: Wafer 단위 다변량 이상 탐지

개별 Wafer가 동시에 여러 Feature에서 99th percentile을 초과하는지 측정합니다. Feature 초과 비율이 5%를 넘는 Wafer를 Outlier로 판정합니다.

3종 Score 산출 과정 및 개념도

Step 3: Feature Importance

각 Score에서 가장 크게 기여한 Feature를 추적합니다. Shift는 방향(악화/개선), Tail은 초과율, Outlier는 공통 초과 Feature를 제공합니다.

Feature Importance: Shift Top-20, Tail Top-20, Shift vs Tail 분류, Multi-Score Heatmap

Step 4: 최종 판정 리포트

최종 판정 대시보드: Score 요약 + 원인 Feature + Outlier Wafer + 권장 액션
