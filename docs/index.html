1. 과제 배경

1.1 문제 정의

반도체 제조 공정에서 ECO(Engineering Change Order)는 수율 개선, 장비 교체, 재료 변경 등 다양한 이유로 적용됩니다. 변경점 적용 후 품질이 기존과 동일한지 검증하는 과정은 필수적이지만, 수천 개의 계측 Feature를 수동으로 비교하는 것은 비효율적이며 간헐적 불량 패턴을 놓칠 위험이 있습니다.

핵심 질문 1

Ref(기준군)와 Compare(변경군) 사이에 얼마나 다른가?

3종 Score(Shift, Tail, Outlier)로 정량화하여 SAFE/CAUTION/RISK/HIGH_RISK 단계로 판정합니다.

핵심 질문 2

어떤 Feature가 Score 변동에 가장 크게 기여하는가?

Score별 Feature Importance를 추적하여 엔지니어에게 구체적인 원인 정보를 제공합니다.

2. 실험 과정

Step 1: 합성 데이터 생성

실제 데이터 투입 전 파이프라인 동작을 검증하기 위해 3가지 불량 패턴을 시뮬레이션한 합성 데이터를 생성했습니다.

그림 2. 합성 데이터 구조 분석 — Ref 1,000장 vs Compare 80장의 비대칭 구조와 3가지 삽입 패턴

Step 2: 전처리 (Preprocessing)

전처리는 3단계로 구성됩니다:

  1. Feature 필터링: 결측률 30% 이상, 분산 ≈ 0인 변수 제거
  2. Robust Scaling: Ref의 median/IQR로 정규화 (compare에도 동일 적용)
  3. Winsorizing: Ref 기준 0.5th~99.5th percentile로 clipping (센서 glitch 제거)

왜 Ref 기준으로 Scaling하는가?

Ref가 baseline이므로, Ref 기준으로 정규화해야 Compare의 이탈 정도가 의미를 가집니다. Compare의 통계량을 사용하면 이탈 자체가 스케일링에 반영되어 차이가 희석됩니다.

4. 토론

4.1 3종 Score 분리의 타당성

이번 PoC에서 Shift Score와 Tail Score의 원인 Feature가 부분적으로 겹치는 것을 확인했습니다(그림 5 좌하단). Pattern A(Systematic Shift)를 삽입한 Feature(EDS_0100~0120)는 전체 분포가 이동했기 때문에 Shift Score와 Tail Score 모두에서 검출됩니다. 이는 "강한 Shift가 있으면 자연스럽게 Tail도 증가한다"는 직관과 일치합니다.

반면 Pattern B(Intermittent Spike)는 중심치는 거의 변하지 않고 소수 wafer에서만 극단값이 발생하므로, 이론적으로 Tail Score에서만 검출되어야 합니다. 합성 데이터에서 이 분리가 확인되었습니다.

Score 분리의 실질적 가치

Shift만 높은 경우: 공정 조건이 전반적으로 바뀐 것 → Recipe/설비 파라미터 점검
Tail만 높은 경우: 간헐적 이상 → 특정 lot/설비/시간대 추적 필요
둘 다 높은 경우: 구조적 변화 → 변경점 원복 고려

4.2 Top-K 방식의 장단점

Shift Score에서 전체 RMS 대신 Top-K(상위 1%) 평균을 사용한 이유는 feature 수가 5,000개일 때 소수 feature의 shift가 전체 RMS에 희석되기 때문입니다. 실험 결과 Top-K 방식이 삽입한 21개 feature의 shift를 정확히 포착했습니다.

다만 top_k_ratio의 선택이 결과에 영향을 미칩니다. 1% = 50개 feature를 보는 것인데, 실제 shift feature가 21개이므로 나머지 29개는 noise를 포함하여 Score가 약간 희석됩니다. 향후에는 adaptive top_k 선택이나 threshold 기반 feature 선택을 고려할 수 있습니다.

4.3 비대칭 데이터의 영향

Ref 1,000장 vs Compare 80장의 비대칭 구조에서 Compare의 통계량 추정은 불안정할 수 있습니다. 민감도 분석(그림 8)에서 확인한 바와 같이, Compare wafer 수가 30 미만이면 Score 변동성이 크게 증가합니다. 30장 이상에서는 비교적 안정적인 Score를 보이며, 80장에서는 충분한 신뢰도를 제공합니다.

4.4 Winsorizing의 효과와 한계

0.5th~99.5th percentile Winsorizing은 센서 glitch나 명백한 measurement 오류를 제거하는 데 효과적입니다. 그러나 Pattern B와 같은 실제 간헐적 불량 신호까지 제거할 위험이 있습니다. 현재 설정(0.5%~99.5%)은 매우 보수적이어서 실제 불량 신호를 보존하면서도 극단적 오류만 제거합니다.

4.5 현실 적용 시 고려사항

고려사항위험대응

설비 편향 (equipment bias) 변경점 아닌 설비 차이가 검출됨 Matched comparison: 동일 설비 ref-comp 매칭

5. 인사이트

3종 Score 분리의 효과

단일 종합 Score로는 놓칠 수 있는 간헐적 불량 패턴(Tail)과 특정 wafer 집중 이상(Outlier)을 별도로 포착할 수 있었습니다. 현실에서 "수율은 괜찮은데 간헐적으로 불량 lot이 나오는" 상황은 Tail Score로만 검출 가능합니다.

Feature Importance의 실용적 가치

Score가 높다는 것만으로는 엔지니어에게 actionable한 정보가 되지 못합니다. "어떤 Feature가 원인인가"를 함께 제공해야 구체적인 공정 조사가 가능합니다. Shift 원인과 Tail 원인이 다를 때 특히 가치가 높습니다.

비대칭 데이터에서의 Robust Scaling

Ref 기준 median/IQR Scaling은 outlier에 강건하면서도 Compare의 이탈 정도를 직관적으로 해석할 수 있게 합니다. Z-score(mean/std) 대비 극단값에 덜 민감하여 소규모 Compare 데이터에서도 안정적입니다.

Sample Size의 중요성

민감도 분석에서 Compare 30장 미만 시 Score 변동성이 급격히 증가함을 확인했습니다. 이는 파이프라인에 내장된 INSUFFICIENT_DATA 판정의 근거가 됩니다. 현장에서 조기 평가 압력이 있더라도, 최소 30장의 Compare를 확보해야 신뢰할 수 있는 판정이 가능합니다.
